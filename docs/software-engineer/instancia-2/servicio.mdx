# Brinda un servicio de excelencia

Escucha los pedidos del cliente interno para responder y actuar en base al pedido, orientándose a generar a acciones concretas. Acude al soporte de un referente ante situaciones que le representan alguna dificultad o bien para casos que sean nuevos.

Tiene la capacidad para analizar y comprender los requisitos del negocio y traducirlos en requisitos técnicos y arquitectónicos para el sistema de software.

**Ejemplos:**
- Responde a preguntas asociadas al funcionamiento, calidad y estado de sus tareas (anteriores o actuales)
- Mantiene informados al cliente interno (persona dentro de la organización, por ejemplo, referente de proyecto) respecto de sus avances en las tareas que realiza.
- Comprende de manera profunda a los usuarios finales del sistema y sus necesidades, preferencias y comportamientos, para diseñar una arquitectura de software que satisfaga eficazmente esas necesidades.
- Tiene una comprensión general de la estrategia de negocio de la organización y cómo el sistema de software contribuye a esa estrategia, ya sea mejorando la eficiencia operativa, impulsando la innovación o generando nuevos ingresos.
