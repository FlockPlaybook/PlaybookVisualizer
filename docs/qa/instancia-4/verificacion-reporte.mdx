# Verificación y reporte

Utiliza todas las herramientas a su alcance para asegurar la integridad y completitud de sus pruebas. Realiza pruebas con una visión íntegra y holística del negocio. 

Analiza los posibles causantes de un error y define, en conjunto con referentes del proyecto, acciones preventivas para evitar que se vuelva a replicar el incidente.

**Ejemplos:**
- Define un conjunto de escenarios a probar ante una situación crítica del proyecto
- Actualiza las condiciones y herramientas de prueba en base a nuevas funcionalidades y/o dispositivos
- Propone priorización en el abordaje de las soluciones
- Indaga acerca de las razones por las cuáles se detectó una falla para evitar que vuelva a suceder el día de mañana y propone acciones preventivas
- Propone nuevas herramientas para incluir en el stack tecnológico
- Evalua entradas, salidas como la lógica interna
- Justifica la importancia de corregir un bug que se intenta desplazar en el tiempo
- Es capaz de adaptarse a herramientas utilizadas por el equipo del cliente
- Capacita a otros en pruebas funcionales como en las estrategias y herramientas asociadas a estas.
- Realiza consultas SQL complejas para la validación de los requerimientos
- Realiza pruebas de accesibilidad y/o SEO
- Evalúa las salidas del código para verificar casos específicos
- Define las prioridades de los distintos tipos de pruebas
- Fundamenta ante clientes internos y externos las decisiones tomadas.
- Considera los diferentes equipos/tecnologías participantes en la definición de su estrategia
- Ejecuta, si el producto lo amerita, pruebas de SEO / Accesibilidad / etc.
