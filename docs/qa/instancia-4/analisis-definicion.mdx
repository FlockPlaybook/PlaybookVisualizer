# Análisis y definición

Define la estrategia de aseguramiento de calidad del proyecto. Colabora en estimaciones comerciales. Comprende en profundidad fundamentos de bases de datos. Detecta necesidad de pruebas no funcionales.

**Ejemplos:**
- Diseñar la estrategia de QA del proyecto (tipos de pruebas, cobertura, criterios de salida).
- Colaborar con el equipo comercial en estimaciones de esfuerzo de QA.
- Usar SQL para consultar bases de datos y validar la integridad de datos.
- Detectar la necesidad de pruebas no funcionales (performance, seguridad, accesibilidad).
- Definir criterios de aceptación de calidad para el proyecto.
