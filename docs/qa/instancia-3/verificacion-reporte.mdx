# Verificación y reporte

Realiza pruebas de dificultad media-alta por fuera de la interfaz gráfica en función de sus sólidas nociones de ingeniería de software. 

Detecta y reporta puntos débiles del sistema, socializa los mismos con el equipo del proyecto y/o cliente.

**Ejemplos:**
- Prioriza y selecciona las tareas pendientes por ejecutar en un plazo establecido (ej: cierre de cards principales al final de un sprint)
- Valida criteriosamente con los diferentes actores involucrados el alcance de los casos de prueba
- Contempla en el proceso de desarrollo la incorporación y actualización de casos de pruebas
- Promueve la validación de casos de prueba por parte del equipo de desarrollo en cada tarea del sprint
- Brinda soporte al equipo de desarrollo para detectar y/o reproducir el error
- Ante la necesidad genera reportes personalizados para clientes internos/externos
- Complementa su reporte con información adicional para colaborar en la solución del error
- Se mantiene informado acerca las nuevas tecnologías, herramientas o estrategias técnicas utilizadas en Desarrollo incluso cuando aún no forma parte del stack tecnológico (Ej.: GraphQL)
- Realiza pruebas de API integrales en forma autónoma. También complementa estas pruebas con validaciones en la base de datos
- Generalmente no requiere soporte de desarrolladores para la ejecución de las pruebas
- Investiga y se capacita en las herramientas necesarias para realizar pruebas funcionales
