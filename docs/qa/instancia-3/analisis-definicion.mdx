# Análisis y definición

Identifica, debate y fundamenta interrogantes de valor. Contempla casos borde del negocio en su totalidad. Empatiza con usuarios potenciales. Identifica posibles vulnerabilidades elementales.

**Ejemplos:**
- Cuestionar y debatir requerimientos con el equipo para identificar puntos ciegos.
- Contemplar casos borde, flujos alternativos y escenarios de error en el plan de pruebas.
- Empatizar con el usuario final para identificar comportamientos no previstos.
- Identificar posibles vulnerabilidades básicas de seguridad en las funcionalidades analizadas.
- Comprender la estructura interna de la API para diseñar pruebas más efectivas.
