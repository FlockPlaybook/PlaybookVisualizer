# Análisis y definición

Analiza requerimientos e identifica flujos críticos de forma autónoma. Elabora documentación de casos de prueba completa. Prioriza y documenta casos adecuadamente.

**Ejemplos:**
- Analizar requerimientos de complejidad media e identificar flujos críticos.
- Elaborar documentación de casos de prueba de forma autónoma y completa.
- Priorizar casos de prueba según el riesgo e impacto de cada funcionalidad.
- Analizar feedback de usuarios finales para incorporarlo al plan de pruebas.
- Identificar dependencias entre casos de prueba y documentarlas.
