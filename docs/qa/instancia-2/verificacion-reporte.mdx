# Verificación y reporte

Realiza pruebas de complejidad baja-media y propone mejoras. 

Comprende el grado de especificidad necesaria para el reporte de hallazgos identificados, siguiendo los lineamientos definidos por el área

**Ejemplos:**
- Interroga a cliente interno o externo acerca de la prioridades en las pruebas
- Valida en el equipo de proyecto los casos de prueba seleccionados
- Prioriza en función de los flujos críticos del sistema
- Considera flujos normales y alternativos en sus pruebas
- Considera casos borde propios del negocio evidentes y casos borde de datos
- Selecciona casos de prueba funcionales, de UI/UX
- Evalúa como es la usabilidad de la aplicación y propone mejoras
- Utiliza una redacción apropiada (impersonal)
- Utiliza terminología del negocio para el reporte de incidencias
- Su reporte es claro y preciso
- Categoriza y prioriza correctamente las incidencias reportadas por terceros
- Realiza pruebas de API con una profundidad limitada
- Complementa sus pruebas con consultas de SQL simples
