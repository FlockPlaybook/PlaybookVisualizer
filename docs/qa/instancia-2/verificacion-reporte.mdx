# Verificación y reporte

Realiza pruebas de complejidad baja-media y propone mejoras. Considera flujos normales y alternativos. Realiza pruebas de API con profundidad limitada.

**Ejemplos:**
- Ejecutar pruebas de complejidad media contemplando flujos alternativos.
- Proponer mejoras a la usabilidad y diseño basadas en su experiencia de testing.
- Realizar pruebas de API básicas usando herramientas como Postman.
- Interrogar al equipo sobre prioridades para orientar el esfuerzo de testing.
- Documentar resultados de ejecución de forma completa.
