# Análisis y definición

Analiza flujos y funcionalidades con guía del referente. Identifica casos de prueba básicos. Solicita datos de prueba pertinentes y consulta ante requerimientos no comprende.

**Ejemplos:**
- Analizar requerimientos de baja complejidad con apoyo del referente.
- Identificar y documentar casos de prueba principales para features simples.
- Solicitar al equipo de desarrollo los datos de prueba necesarios.
- Hacer preguntas al referente cuando los requerimientos no son claros.
- Documentar los casos de prueba en la herramienta del proyecto (Jira, TestRail, etc.).
