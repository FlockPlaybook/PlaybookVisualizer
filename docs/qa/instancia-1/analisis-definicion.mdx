# Análisis y definición

Analiza los flujos y funcionalidades más importantes del sistema con la guía de su referente

**Ejemplos:**
- Identifica casos de prueba asociados a un requerimiento o funcionalidad
- Estructura las user stories/requisitos basándose en el template definido
- Solicita datos de pruebas pertinentes:
- No utiliza datos sin sentido para el escenario a validar
- Los datos utilizados representan información que podría utilizarse en el uso productivo del       sistema
- Varía sus datos de prueba para evaluar escenarios diversos
- Obtiene credenciales de acceso adecuadas para los casos de prueba a validar
- Procura contar con un ambiente para realizar las pruebas y se asegura que el mismo esté explícito en las tareas a probar
- Busca definición sobre delimitación y tipo de dato de la información a trabajar por la aplicación
- Realiza consultas ante requerimientos que no comprende en su totalidad
