# 📊 Verificación y reporte — Evolución por instancia

> Cómo crece esta responsabilidad a lo largo de las instancias.

---

## Instancia 1

Realiza pruebas en sistemas simples desde la interfaz gráfica. Su reporte de errores es claro y permite reproducir el defecto.

**Ejemplos:**
- Ejecutar casos de prueba manuales sobre la interfaz gráfica.
- Reportar defectos de forma clara, incluyendo pasos para reproducir, resultado esperado y actual.
- Verificar la corrección de defectos tras el fix del desarrollador.
- Documentar el resultado de cada ejecución de prueba.

---

## Instancia 2

Realiza pruebas de complejidad baja-media y propone mejoras. Considera flujos normales y alternativos. Realiza pruebas de API con profundidad limitada.

**Ejemplos:**
- Ejecutar pruebas de complejidad media contemplando flujos alternativos.
- Proponer mejoras a la usabilidad y diseño basadas en su experiencia de testing.
- Realizar pruebas de API básicas usando herramientas como Postman.
- Interrogar al equipo sobre prioridades para orientar el esfuerzo de testing.
- Documentar resultados de ejecución de forma completa.

---

## Instancia 3

Realiza pruebas de dificultad media-alta fuera de la interfaz gráfica. Detecta y reporta puntos débiles del sistema. Realiza pruebas de API integrales en forma autónoma.

**Ejemplos:**
- Ejecutar pruebas complejas incluyendo pruebas de API end-to-end.
- Detectar y reportar debilidades del sistema más allá de los casos documentados.
- Realizar pruebas de integración entre componentes del sistema.
- Automatizar casos de prueba de regresión básicos.
- Proponer mejoras al flujo de reporte y seguimiento de defectos.

---

## Instancia 4

Visión íntegra y holística del negocio en la verificación. Define acciones preventivas. Realiza consultas SQL complejas. Realiza pruebas de accesibilidad y/o SEO.

**Ejemplos:**
- Ejecutar pruebas con visión completa del negocio, anticipando impactos transversales.
- Definir acciones preventivas para mitigar riesgos de calidad antes de que ocurran.
- Realizar consultas SQL complejas para validar la integridad de datos en el backend.
- Ejecutar auditorías de accesibilidad (WCAG) y SEO técnico.
- Liderar la verificación de releases críticos con el equipo.
