# Calidad de los entregables

Procura mantener altos estándares de calidad en el código y las soluciones desarrolladas.

Colabora identificando fallas de diseño en los desarrollos existente.

Propone modificaciones con el objetivo de lograr mejoras en la performance, seguridad, mantenibilidad y escalabilidad del software.  

Conoce e implementa con el equipo estándares de codificación, algoritmia y las mejores prácticas para asegurar la coherencia y calidad del código.

Implementa pruebas unitarias y automatizadas para garantizar la estabilidad y fiabilidad del software.

Supervisa la gestión efectiva de defectos y problemas técnicos. Trabaja con el equipo para identificar, priorizar y abordar rápidamente los problemas que afectan la calidad del software.

Contribuye en la definición de métricas de calidad y participa en la monitorización de la salud del sistema.

**Ejemplos:**
* Utiliza herramientas de análisis estático de código para identificar y corregir posibles violaciones de estándares, asimismo comparte su conocimiento en la utilización de estas herramientas para el resto del equipo.
* Interpretar informes de calidad de código y toma medidas para mejorar los aspectos señalados.
* Comprende y aplica las lecciones aprendidas durante las revisiones de código en sus futuros desarrollos.
* Contribuye a la creación de documentación de diseño simple, resaltando las decisiones arquitectónicas y la estructura general del sistema.
* Contribuye con ideas para mejorar los procesos de desarrollo, enfocándose en la calidad del código.
* Utiliza y comparte el conocimiento de prácticas y herramientas de code review para mejorar la calidad del código y asegurar el cumplimiento de estándares de codificación.
