# Diseño de soluciones

Diseña una estrategia de implementación (solución) para aplicaciones de complejidad alta involucrando conceptos de testing continuo que le permiten abarcar todos los aspectos relacionados a la calidad del proyecto.

Participa de estimaciones comerciales para aplicaciones de complejidad alta de manera autónoma.

**Ejemplos:**
- Diseña estrategias para aplicaciones con arquitecturas complejas como ser: microservicios, microfrontends, etc
- Toma al testing contínuo como una cultura de trabajo y plasma los conceptos relacionados en la estrategia definida.
- Es capaz de tomar decisiones sobre cuándo conviene usar un lenguaje/framework u otro para un proyecto, considerando beneficios y limitantes de una herramienta/framework sobre otro y/o la necesidad del proyecto.
- Es capaz de sugerir, a alto nivel, implementaciones complejas o críticas de requerimientos, como también la arquitectura o infraestructura adecuada.
- Utilizando su experiencia y conocimiento, tiene la capacidad de estimar propuestas comerciales, logrando dar un estimado con un error aceptable (dentro de los parámetros esperados por sus referentes) sin contar con el detalle del proyecto.
