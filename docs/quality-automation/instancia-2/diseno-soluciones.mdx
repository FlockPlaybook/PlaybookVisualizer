# Diseño de soluciones

Diseña una estrategia de implementación (solución) para aplicaciones de complejidad media-baja considerando la relación costo/beneficio de la implementación para cumplir con la expectativa de calidad esperada por el proyecto. 

Identifica responsabilidades y alcance de cada tipo de pruebas automatizadas o pruebas de performance, necesidades de desarrollo y arquitectura para el cumplimiento de la estrategia definida. 

Participa en la definición y diseño de estimaciones comerciales.

**Ejemplos:**
- Diseña estrategias para proyectos con arquitecturas tradicionales de Cliente-Servidor
- Utiliza como guía o lineamiento la pirámide de Cohn para la aplicación de los distintos tipos de automation sin caer en prácticas inadecuadas (Ej.: hacer gran cantidad tests de Interfaz de usuario y una cantidad reducida de test de servicios).
- Prioriza y selecciona autónomamente qué se automatiza
- Identifica flujos/funcionalidades automatizables y no automatizables
- Comunica y busca consenso de la estrategia definida
- Considera los diferentes equipos participantes en la definición de su estrategia
- Define la instancia de ejecución y el conjunto de pruebas automatizadas, en la herramienta de integración continua definida para el proyecto o por el área.
- Comprende el sistema como un todo, pudiendo entender el impacto de los cambios en los sistemas y revisando posibles acciones a tomar con quién gestiona el proyecto para mitigar potenciales riesgos en caso de haberlos.
- Enfatiza su análisis en aspectos críticos del sistema e identifica claramente la cuestiones superficiales.
- Mantiene comunicación fluida con otros actores intervinientes en el proyecto con el objetivo de realizar un análisis abarcativo del sistema.
- Es capaz de comparar librerías/frameworks/herramientas y definir la/el indicada/o para un proyecto.
- Puede estimar propuestas comerciales de complejidad media/baja de manera autónoma y de complejidad alta con soporte de su referente.
