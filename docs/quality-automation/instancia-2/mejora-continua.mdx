# Mejora continua

Ayuda a otros a mejorar la calidad de su implementación, asegurando el cumplimiento de estándares y atributos de calidad de código dentro de su proyecto.

Toma responsabilidad por la implementación de objetivos estratégicos para el área.

**Ejemplos:**
- Toma tareas propuestas por sí mismo para mejorar la calidad técnica del proyecto
- Realiza revisiones de código valiosas para los contribuidores, ya sea porque aporta a la calidad del código y al cumplimiento de los lineamientos establecidos, como así también al aprendizaje de quien lo recibe.
- Puede desarrollar la base (bootstrap) de un framework desde el inicio o sugerir/realizar modificaciones de valor de forma autónoma.
- Toma ownership y responsabilidad en tareas de investigación o pruebas (PoC) para la implementación de nuevas técnicas o tecnologías y genera documentación para la transmisión de conocimiento al equipo.
- Busca constantemente mejorar la calidad técnica de sus entregables.
