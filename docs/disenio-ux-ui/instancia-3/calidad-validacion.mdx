# Calidad y validación

Participa activamente en las instancia de PDU en compañia de otros miembros del equipo.

Comprende la mayor parte de los entregables de los cuales que es responsable. 

Conoce y aplica con mayor frecuencia los parámetros de accesibilidad, se apoya en su referente técnico cuando sea necesario. 

Comienza a considerar las métricas que impactan en el producto para tomar decisiones más robustas.

**Ejemplos:**
- Se familiariza con herramientas como Mapas de empatía, viajes de usuario, etc para encontrar insights y definir usuarios objetivo.
- Participa en sesiones de ideación junto con clientes o persona usuarias.
- Utiliza herramientas para chequear contrastes de color en etapas de diseño (ej. Web Aim).
- Pide y lleva adelante el proceso de Design Review.
