# Calidad y validación

Tiene un manejo muy fluido en las instancias de PDU, pudiendo lleva a cabo pruebas de usuario comparativas o sucesivas para productos en desarrollo o en producción. 

Genera hipótesis sobre productos de mercado ya existentes: sugerir un plan de acción en base a investigación.

Puede realizar informes de accesibilidad sobre productos en producción, definiendo el criterio de evaluación y sugiriendo maneras de resolver los problemas.

**Ejemplos:**
- Entiende el proceso con profundidad, pudiendo adaptarse a las necesidades del proyecto, sin dejar de cumplir con los requisitos del proceso.
- Guia al equipo en la toma de decisiones a partir de hallazgos, basados en datos, garantizando que no se trabaje con insights aislados o fuera de contexto.
- Asesora al cliente sobre oportunidades de levantamiento de datos. (ej. Mapas de calor, analytics, etc)
- Cuestiona la objetividad de los datos analizados.
- Tiene nociones de los tags de HTML relacionados a accesibilidad.
