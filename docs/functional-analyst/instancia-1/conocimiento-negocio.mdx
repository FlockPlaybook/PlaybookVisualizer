# Conocimiento de negocio y estrategia de producto

Comprende el negocio del cliente y lo transmite al equipo. Planifica tareas a corto plazo. Realiza seguimiento continuo del Product Backlog.

**Ejemplos:**
- Comprender el modelo de negocio del cliente y comunicarlo al equipo de desarrollo.
- Planificar y organizar el backlog para el sprint próximo.
- Hacer seguimiento de las tareas del sprint y comunicar el estado al equipo.
- Participar en reuniones con el cliente para capturar necesidades de negocio.
