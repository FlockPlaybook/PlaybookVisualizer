# Negocio, producto y transferencia de valor

Diseña validaciones problema-solución, define indicadores de adopción y criterios de MVP, y anticipa riesgos con planes de mitigación. Evalúa el aporte al portfolio y la factibilidad de productización, y participa en la planificación estratégica y la preventa con base tecnológica.

**Ejemplos:**

- Define qué métrica de adopción probaría que la solución vale la pena (ej.: % de usuarios que la usan por semana).
- Estima si la solución se puede volver producto o si solo sirve para un cliente puntual.
- Define los criterios de un MVP y qué quedaría fuera del primer release.
- Aporta el componente técnico a una propuesta de preventa.

<VerticalExamples>

**Soluciones inmersivas y avatares**

- Define indicadores de adopción y criterios de MVP del avatar.
- Estima si el avatar puede volverse producto o es a medida.
- Aporta el componente técnico a una propuesta de avatar.

**Computer Vision**

- Define métricas de adopción y MVP de una solución de CV.
- Evalúa si el modelo escala a otros clientes.
- Define qué queda fuera del primer release de visión.

**Robótica**

- Define adopción y MVP de una solución robótica por industria.
- Estima si el caso robótico es comercializable.
- Aporta la parte técnica de una propuesta de robótica.

**Sistemas multi-agent**

- Define adopción y MVP de una solución multi-agent.
- Evalúa si el flujo se puede productizar o es a medida.
- Define el costo por tarea que hace viable el negocio.

**Desarrollo de productos comerciales**

- Valida problema-solución y define criterios de MVP del producto.
- Define la métrica de adopción que justifica invertir.
- Aporta el componente técnico a una preventa.
