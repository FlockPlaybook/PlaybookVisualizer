# Validación, métricas y evidencia

Ejecuta validaciones simples previamente definidas y compara los resultados contra criterios acordados. Detecta inconsistencias, regresiones o diferencias entre corridas y las comunica al equipo. Registra métricas básicas como precisión, latencia, tasa de error o cumplimiento de casos esperados. Puede proponer casos adicionales de prueba, pero todavía no define de manera autónoma la estrategia de validación.

**Ejemplos:**

- Ejecuta el set de validación definido y compara los resultados con el umbral acordado.
- Detecta que dos ejecuciones con la misma entrada producen resultados diferentes y documenta la inconsistencia.
- Propone agregar un caso de prueba que no estaba contemplado.
- Registra las métricas básicas (precisión, latencia, tasa de error) de cada corrida.

**Soluciones inmersivas y avatares**

- Compara la latencia entre motores de TTS/STT y detecta regresiones.
- Verifica los resultados contra el umbral acordado de latencia.
- Arma un set de pruebas para correr tras cada cambio.

**Computer Vision**

- Mide precisión y recall básicos y crea pruebas de regresión del dataset.
- Compara los resultados contra el umbral acordado.
- Detecta que dos corridas iguales dan métricas distintas.

**Robótica**

- Mide el error de navegación y detecta inconsistencias entre corridas.
- Compara la trayectoria real contra la esperada.
- Crea un set de recorridos de regresión.

**Sistemas multi-agent**

- Mide la calidad de las salidas de los agentes y detecta inconsistencias.
- Compara el resultado contra el criterio acordado.
- Registra el costo y la latencia por conversación.

**Desarrollo de productos comerciales**

- Mide indicadores de uso básicos y crea pruebas de regresión.
- Compara el comportamiento contra el criterio esperado.
- Documenta una inconsistencia entre dos ejecuciones.
