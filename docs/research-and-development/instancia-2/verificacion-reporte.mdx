# Verificación y reporte

Realiza pruebas de complejidad media, diseñando y ejecutando casos de prueba complejos y exhaustivos para identificar posibles errores o fallos.

Propone soluciones y mejoras significativas basadas en los hallazgos de las pruebas realizadas, contribuyendo activamente al proceso de mejora continua del producto.

Produce informes detallados y completos sobre los resultados de las pruebas, utilizando herramientas y métricas específicas para comunicar eficazmente los hallazgos.

**Ejemplos:**
- Registra de los resultados de las pruebas realizadas en un formato definido por el equipo de QA o testing.
- Asiste a reuniones de revisión para discutir los hallazgos de las pruebas realizadas y ofrecer aportes en la identificación de soluciones.
- Crea pruebas de regresión para validar la estabilidad del sistema ante cambios.
- Genera de informes detallados con recomendaciones para mejorar la calidad del software
- Evalua nuevas herramientas o métodos de pruebas que podrían mejorar el proceso de verificación.
- Describe de manera clara y concisa los problemas encontrados siguiendo una estructura predeterminada
